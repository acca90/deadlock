Deadlock Problem

In computer science according to Tenanbaum, there is a problem known as DeadLock. “A conjunct of process will be in a Deadlock situation if all process that is from a conjunct are waiting for an event that only another process that is from the same conjunct could do, but as it is locked waiting the event, that event will never occur because the process will only generate the event when the other process finish.” For that to happen there is some conditions.

  • Mutual Exclusion: every resource is either associated with a single process or available;
  • Possession and Waiting: processes that hold resources can request new resources;
  • No Preemption: resources previously granted cannot be taken;
  • Circular wait: there must be a circular chain of two or more processes, in which each is waiting for resources retained by the next member of that chain.

A Deadlock that can happen physically is on the traffic, this occurs when the vehicles block the passage of others waiting for their moment to continue, but so do they, locking the traffic in every way.
